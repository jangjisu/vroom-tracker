# 출발지·도착지 선택 설계

## 목적

사용자가 현재 위치만 출발지로 사용해야 하는 제약을 없애고, 출발지와 도착지를 장소 검색 또는 지도에서
직접 선택하게 한다. 검색 결과가 여러 개면 첫 결과를 자동 사용하지 않고 사용자가 명시적으로 선택한다.

## 범위

- Frontend만 변경한다.
- 기존 `/api/place-search`와 `/api/route-rest-stops` 계약을 그대로 사용한다.
- 역지오코딩과 새로운 외부 API는 추가하지 않는다.
- 출발지는 현재 페이지가 열려 있는 동안 유지하며 영구 저장하지 않는다.

## 사용자 흐름

### 출발지 설정

1. 출발지가 없는 상태에서 사용자가 출발지 설정 또는 경로 찾기를 누르면 출발지 선택창을 연다.
2. 사용자는 `현재 위치 사용`, `장소 검색`, `지도에서 선택` 중 하나를 고른다.
3. 현재 위치는 브라우저 위치 권한을 요청하고 성공한 좌표를 출발지로 확정한다.
4. 장소 검색은 후보의 이름과 주소를 목록으로 보여주며, 사용자가 고른 후보를 출발지로 확정한다.
5. 한 번 확정한 출발지는 다른 목적지를 검색해도 유지하고, `출발지 변경`을 눌렀을 때만 바꾼다.

### 도착지 설정

1. 사용자는 장소 검색 또는 지도 선택으로 도착지를 정한다.
2. 장소 검색 결과는 첫 항목을 자동 선택하지 않고 후보 목록을 표시한다.
3. 후보를 고르면 도착지 이름과 좌표를 저장하되 경로 조회는 즉시 실행하지 않는다.

### 지도 선택

1. 출발지 또는 도착지의 `지도에서 선택`을 누르면 지도 선택 모드로 전환한다.
2. 지도 클릭 위치에 임시 핀을 표시하고, 다른 위치를 누르면 임시 핀을 이동한다.
3. `이 위치로 선택`을 눌러야 해당 좌표를 확정한다.
4. `취소` 또는 `Escape`를 누르면 임시 핀을 제거하고 기존 선택값을 유지한다.
5. 역지오코딩을 하지 않으므로 확정된 위치명은 `지도에서 선택한 위치`로 표시한다.

### 경로 조회

1. 출발지와 도착지가 모두 확정된 상태에서 `경로 휴게소 찾기`를 누른다.
2. 기존 경로 API에 두 지점의 위도·경도와 도착지명을 전달한다.
3. 기존 경로 폴리라인, 출발·도착 마커와 경로상 휴게소 목록을 그대로 표시한다.

## 화면 구성

- 기존 단일 목적지 검색 영역을 출발지 요약, 도착지 검색과 경로 찾기 버튼으로 재구성한다.
- 출발지 요약은 현재 선택값과 `변경` 동작을 제공한다.
- 도착지는 검색어 입력, 검색, 지도 선택 동작을 제공한다.
- 출발지 선택창은 현재 위치, 장소 검색과 지도 선택 진입점을 제공한다.
- 장소 후보 모달은 출발지와 도착지 양쪽에서 재사용하며 제목과 선택 콜백만 대상에 맞게 바꾼다.
- 지도 선택 모드에서는 안내 문구, `이 위치로 선택`, `취소`를 지도 위에 표시한다.

## 구성 요소와 책임

### `route-point-selection.js`

- 출발지, 도착지, 현재 후보 검색 대상과 지도 선택 대상을 관리한다.
- 좌표 유효성, 지점 확정, 선택 취소와 경로 조회 가능 여부를 순수 함수 중심으로 제공한다.
- DOM, 네이버 지도와 `fetch`에 의존하지 않는다.

### `rest-stops-map.js`

- 입력과 모달 이벤트를 연결하고 선택 상태를 화면에 반영한다.
- 브라우저 위치 조회, 네이버 지도 클릭과 임시 마커 생명주기를 관리한다.
- 장소 검색 결과를 현재 검색 대상에 맞춰 출발지 또는 도착지로 확정한다.
- 두 지점이 준비된 경우 기존 `routeRequest.load`를 호출한다.

### `index.html`과 `style.css`

- 출발지·도착지 입력 영역, 출발지 선택창과 지도 선택 도구의 구조와 스타일을 제공한다.
- 기존 경로 결과, 휴게소 상세와 지도 레이아웃은 유지한다.

## 상태와 오류 처리

- 위치 권한 거부 또는 위치 조회 실패 시 출발지를 확정하지 않고 안내 메시지를 표시한다.
- 장소 검색 실패와 외부 API 장애는 기존 공통 alert와 경로 상태 메시지를 사용한다.
- 후보가 없으면 현재 선택값을 유지한다.
- 지도 선택 취소, 모달 닫기와 `Escape`는 기존 출발지·도착지를 변경하지 않는다.
- 출발지나 도착지가 없으면 API를 호출하지 않고 누락된 지점의 설정 UI를 연다.
- 새 선택을 확정한 뒤에도 기존 경로는 다음 경로 조회 전까지 유지한다.

## 테스트

- 출발지와 도착지 상태가 독립적으로 저장되는지 검증한다.
- 현재 위치, 검색 후보와 지도 좌표가 동일한 지점 형식으로 정규화되는지 검증한다.
- 지도 임시 선택을 취소하면 기존 값이 유지되는지 검증한다.
- 두 지점이 모두 있어야 경로 조회 가능 상태가 되는지 검증한다.
- 장소 후보 선택 대상에 따라 출발지 또는 도착지만 변경되는지 검증한다.
- 기존 장소 검색, 경로 요청, 지도 팝업과 상세 화면 테스트가 계속 통과하는지 확인한다.

## 완료 조건

- 출발지를 현재 위치, 장소 후보 또는 지도 좌표로 설정할 수 있다.
- 도착지를 장소 후보 또는 지도 좌표로 설정할 수 있다.
- 검색 후보 첫 항목을 자동 선택하지 않는다.
- 지도 위치는 임시 핀을 확인한 뒤에만 확정된다.
- 출발지는 사용자가 변경하기 전까지 페이지 세션 동안 유지된다.
- 기존 경로와 휴게소 상세 기능에 회귀가 없다.
