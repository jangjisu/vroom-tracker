# vroom-tracker 개발 규칙

## 작업 수행 방식

요청을 받으면 아래 순서로 진행한다.

1. 요청을 수행하기 위해 필요한 규칙 카테고리를 파악한다 (Git / API연동 / Java / 프론트엔드)
2. 필요한 카테고리가 2개 이상이면, 어떤 것부터 진행할지 사용자에게 먼저 질의한다
3. 사용자가 확인한 카테고리 하나에 대한 작업만 수행한다
4. 완료 후 다음 카테고리를 이어서 할지 사용자에게 확인한다

> 카테고리 내부에서 여러 파일을 수정하는 것은 허용된다.
> 단, 카테고리를 넘나들며 동시에 수행하지 않는다.

---

## 규칙 파일 참조

**Git 커밋 관련 요청인 경우**
→ `rules/git-commit.md` 를 읽어라

**새 API 연동 관련 요청인 경우 (ExApiClient, 응답 VO, API.md 수정 포함)**
→ `rules/api-integration.md` 를 읽어라

**Java 파일 수정 관련 요청인 경우**
→ `rules/backend/index.md` 를 읽어라

**JS, HTML, CSS 파일 수정 관련 요청인 경우**
→ `rules/frontend.md` 를 읽어라
