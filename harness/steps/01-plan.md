# 1단계: 계획 생성

목표: 이번 run이 어떤 작업인지, 어디까지 진행할 것인지 정한다.

계획 생성 단계에서는 다음을 확인한다.

- 작업 유형
- 작업 범위
- 관련 기준 문서
- 계획 문서

이 단계에서 답해야 하는 질문은 다음과 같다.

- 사용자가 요청한 변경은 무엇인가?
- 이번 run에서 먼저 진행할 카테고리는 무엇인가?
- 어떤 규칙 파일을 참고해야 하는가?
- `PRODUCT.md`, `DATA.md`, `ARCHITECTURE.md`, `QUALITY.md` 중 무엇이 영향을 받는가?
- 어떤 파일들이 변경될 것으로 예상되는가?

작업 범위가 불명확하거나 여러 카테고리가 섞여 있다면, 다음 단계로 넘어가기 전에 사용자 확인이 필요하다.

기능의 목적이나 동작이 불명확하면 Superpowers의 brainstorming 방식으로 한 번에 하나씩 질문하고, 설계 승인 전에는 구현하지 않는다.
