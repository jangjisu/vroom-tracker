# 2단계: 스펙 영향도 확인

목표: 이번 변경이 API 스펙 또는 외부 연동 스펙에 영향을 주는지 확인한다.

백엔드 작업은 스펙이 실제로 바뀌지 않더라도 이 단계를 지나야 한다.

중요한 기준은 다음과 같다.

```text
스펙 확인은 항상 한다.
스펙 변경은 필요할 때만 한다.
```

이 단계에서 답해야 하는 질문은 다음과 같다.

- 우리 서버가 제공하는 API 스펙이 바뀌는가?
- 외부 공공 API 연동 스펙이 바뀌는가?
- 바뀐다면 endpoint, request, response, error 중 무엇이 바뀌는가?
- 바뀌지 않는다면 왜 바뀌지 않는다고 판단했는가?

DTO, 예외 처리, 응답 코드, Controller 변경은 스펙 영향 가능성이 높으므로 특히 주의한다.
