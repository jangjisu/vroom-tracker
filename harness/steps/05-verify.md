# 5단계: 검증

목표: 테스트와 검증을 실행하고, 검증이 실제로 수행되었다는 증거를 남긴다.

검증 단계에서는 다음을 확인한다.

- 전체 테스트를 실제로 실행했고 성공했는가
- JaCoCo 전체 line coverage 95% 기준을 만족했는가
- 신규·변경 실행 라인이 테스트에서 모두 실행되었는가
- 변경된 핵심 조건의 의미 있는 분기를 모두 검증했는가

백엔드 작업에서는 기본적으로 `./gradlew test` 와 `./gradlew jacocoTestCoverageVerification` 실행을 검증 대상으로 본다.

프론트엔드, 화면 변경, 외부 API 실제 호출 검증 등은 필요하다고 판단될 때 별도 hook으로 추가한다.

JaCoCo XML과 Git 변경 라인을 연결해 신규·변경 실행 라인 및 해당 조건 분기의 100% 커버리지를 검사한다.

자동 검증이 통과하면 Compound Engineering의 코드 리뷰로 구현 범위, 오류 가능성과 테스트 의미를 최종 확인한다.
재사용할 가치가 있는 문제 해결이나 설계 결정이 있다면 Compound 문서로 남기고, 단순 작업이면 생략 이유를 기록한다.
