# 품질 방향

이 문서는 테스트, 정적 분석과 코드 리뷰의 품질 기준을 기록한다.

## 현재 상태

- Checkstyle, Spotless, Palantir Java Format과 ESLint를 사용한다.
- 전체 테스트는 Gradle로 실행한다.
- JaCoCo 전체 라인 커버리지 최소 기준은 95%다.
- 변경된 주요 Java 클래스에 대응하는 테스트 파일 존재 여부를 하네스가 확인한다.

## 합의된 목표

- 신규 및 변경된 실행 라인은 테스트에서 100% 실행한다.
- 변경된 핵심 조건의 true/false 등 의미 있는 분기는 100% 검증한다.
- 프로젝트 전체 커버리지는 기존 기준보다 낮아지지 않게 유지하고 점진적으로 높인다.
- 커버리지 수치를 채우기 위한 의미 없는 테스트는 작성하지 않는다.
- 정상 흐름뿐 아니라 실패 경로, null, 빈 결과와 경계값을 검증한다.
- 테스트는 실행 여부가 아니라 의미 있는 assertion으로 동작을 증명해야 한다.
- 설정 클래스, 단순 접근자와 생성 코드 등은 제외 이유가 분명할 때만 커버리지 대상에서 제외한다.

## 하네스 적용 상태

- 전체 테스트와 전체 라인 커버리지 95% 기준을 자동 검사한다.
- JaCoCo XML과 Git diff를 연결해 신규·변경 실행 라인 100%를 자동 검사한다.
- 변경된 조건문에 JaCoCo 분기 정보가 있으면 미검증 분기가 없는지 자동 검사한다.
- 구현이 끝나면 자동 검증 전에 코드, 테스트와 관련 Markdown 기준 문서의 정합성을 한 번만 리뷰한다.
- 리뷰 결과에는 `문서 정합성` 또는 `Documentation consistency` 제목을 포함한다.
- 리뷰 지적을 반영한 뒤 재리뷰하지 않고 관련 테스트와 전체 자동 검증을 수행한다.

## 미결정 사항

- 전체 프로젝트 커버리지 기준을 올리는 주기
- Mutation testing 도입 여부
