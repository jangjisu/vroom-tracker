# 제품 방향

이 문서는 `rest-route`가 현재 무엇을 만들고 있으며, 기능 판단에서 무엇을 우선하는지 기록한다.
사용자 유형과 아직 검증되지 않은 행동 가설은 `USER_INSIGHTS.md`에서 관리한다.

## 현재 상태

- 전국 고속도로 휴게소 위치를 지도에서 탐색하고 현재 위치를 확인할 수 있다.
- 휴게소 이름으로 직접 검색해 찾을 수 있고, 이름이 같은 후보가 여러 개면 목록에서 골라 바로 상세 정보로 이동할 수 있다.
- 휴게소별 위치, 편의시설, 운영 상태, 주차, 먹거리와 주유 정보를 상세 화면에서 확인할 수 있다.
- 관리자는 휴게소 대표 이미지를 등록·교체·삭제할 수 있고, 등록된 이미지는 사용자 상세 화면과 경로 선택 목록에 용도별 크기로 표시된다. 이미지가 없는 휴게소는 기존 정보 중심 레이아웃을 유지한다.
- 관리자는 휴게소별 먹거리 메뉴를 직접 추가하거나, 실제로 가봤을 때 값이 다른 동기화 메뉴를 직접 고칠 수 있다(수정한 메뉴는 이후 자동 동기화에서 제외된다). 메뉴별 이미지도 등록할 수 있으나, 사용자 화면에 어떻게 노출할지는 아직 정하지 않아 이번 범위에서는 표시되지 않는다.
- 목적지 검색 후보를 직접 선택하고, 현재 위치부터 목적지까지의 경로와 경로상 휴게소를 확인할 수 있다.
- 휴게소 데이터는 서버 시작과 정기 스케줄로 수집하며, 주유 가격은 단건 갱신도 지원한다.

## 합의된 목표

- 현재 제품 범위는 고속도로 휴게소 정보 서비스다.
- 사용자는 이동 중 가까운 휴게소를 찾는 것보다, 이동 경로에 있는 휴게소를 비교해 들를 곳을 고른다.
- 지도에서 휴게소를 탐색하고 상세 정보를 비교하는 기능을 계속 개선한다.
- 먹거리, 입점 매장, 대표 메뉴, 휴게소 특색처럼 방문 결정을 돕는 정보를 우선한다.
- 관리자가 등록하는 대표 이미지는 휴게소의 분위기와 특색을 보여 주어 방문 결정을 돕는 정보로 취급한다.
- 편의시설, 주차 규모, 정비 여부와 같은 실용 정보도 함께 제공한다.
- 현재 위치와 목적지를 기준으로 한 경로 탐색을 출발지 직접 설정과 휴게소 비교 기능으로 확장한다.
- 미래 확장을 예상해 현재 필요하지 않은 계층이나 추상화를 미리 만들지 않는다.

## 데이터 확장 원칙

- 새로운 정보는 사용자가 외부 API 명세를 제공하거나, 사용자와 데이터 출처를 함께 조사한 뒤 추가한다.
- 확인되지 않은 정보는 추정하거나 임의로 생성하지 않는다.
- 관리자 등록, 사용자 후기 등 외부 API 이외의 데이터 수집 방식은 필요해질 때 별도로 결정한다.

## 미결정 사항

- 휴게소 비교와 추천 기준
- 사용자 저장, 후기, 방문 기록 기능의 도입 여부
- 지도에서 출발지와 목적지를 직접 선택할 때 사용할 확정 방식
